# Convenciones de Git

Usaremos la convención [Conventional Commits](https://www.conventionalcommits.org/es/v1.0.0/).

## Prefijos de commits

- `feat:` → nueva funcionalidad
- `fix:` → corrección de errores
- `docs:` → cambios en la documentación
- `style:` → cambios de formato (espacios, comas, etc.)
- `refactor:` → cambios en el código sin afectar la funcionalidad
- `test:` → agregar o modificar pruebas
- `chore:` → tareas de mantenimiento (build, dependencias, configuraciones)

## Ejemplos

- `feat(home-hero)`: agregar sección hero en la página de inicio
- `feat(navbar)`: implementar barra de navegación responsive
- `fix(footer)`: corregir enlaces rotos en el footer
- `docs(setup)`: actualizar instrucciones de instalación en el README
- `style(buttons)`: mejorar estilos de botones en toda la web
- `refactor(layout)`: reorganizar la estructura del layout principal
- `test(contact-form)`: añadir pruebas unitarias al formulario de contacto
- `chore(deps)`: actualizar dependencias de Next.js y Tailwind
